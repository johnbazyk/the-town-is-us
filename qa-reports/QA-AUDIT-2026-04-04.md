| Component | CategoryScale | LinearScale | PointElement | LineElement | BarElement | ArcElement | Filler |
|---|---|---|---|---|---|---|---|

| AcademicPerformance.jsx | ✅ | ✅ | ✅ | ✅ | ✅ | ❌ | ❌ |
| PropertyTaxRatchet.jsx | ✅ | ✅ | ✅ | ✅ | ✅ | ❌ | ❌ |
| DemographicsHousing.jsx | ✅ | ✅ | ✅ | ✅ | ✅ | ❌ | ✅ |
| SchoolClimate.jsx | ✅ | ✅ | ❌ | ❌ | ✅ | ❌ | ❌ |

**Missing registrations:**

- **AcademicPerformance.jsx** — missing `ArcElement`, `Filler`. Low risk if no pie/doughnut charts or area fills are used in this component.
- **PropertyTaxRatchet.jsx** — missing `ArcElement`, `Filler`. Same caveat.
- **DemographicsHousing.jsx** — missing `ArcElement`. Risk: if any pie chart is used here, it will throw a runtime error. `Filler` is present (area charts).
- **SchoolClimate.jsx** — missing `PointElement`, `LineElement`, `ArcElement`, `Filler`. Risk: if this component renders line charts, it will fail at runtime. Only `BarElement` is registered.

**Recommendation:** Either (a) add the missing registrations to each component defensively, or (b) create a shared `src/chartSetup.js` that registers all Chart.js components once and import it in `src/main.jsx`. Option (b) is cleaner and prevents future drift.

**Note:** Missing registrations only cause runtime errors if the unregistered element type is actually rendered. The build passes because Chart.js registration errors are runtime-only. Manual browser testing on each chart tab is needed to confirm no console errors occur.
